# 问题 1.2：1D ResNet 主干网络与 MQAH 模块之间的关系

## 问题

主干网络与 MQAH 之间的关系说明得不够清楚，包括特征维度如何匹配，以及注意力图在平均和上采样之后会带来什么影响。

## 回复

感谢审稿人提出这一重要意见。我们同意原稿对共享 1D ResNet 主干网络与 MQAH 模块之间的数据流，以及最终解释图的生成过程说明得还不够清楚。为回应这一问题，我们在修订稿中将该部分改写为更明确的张量级描述。具体而言，输入 ECG 先经过共享的 1D ResNet 主干网络，得到长度压缩后的特征序列 `X_feat ∈ R^(B×C×L')`。对于本文 PTB-XL 的主实验设置，我们在正文中补充说明了一个具体示例：输入信号形状为 `(B, 12, 1000)`，主干网络输出对应的压缩时间长度为 `L'=32`。随后，每个类别特异性的 MQAH 头在该特征序列上进行 key/value 投影，并与该类别对应的多组可学习 query 向量进行注意力计算，从而生成该类别的预测 logit 及其多查询注意力图。

我们进一步澄清，多查询注意力图上的平均操作仅用于将同一类别下多个 query 对不同形态学证据的关注整合为单一的类别级解释图，从而提高可读性。该平均操作并不是新的推理模块，也不改变模型的决策路径；它的作用是将多个 query 的响应汇总为一个更易于展示和分析的类别级热图，因此相较于单个 query 的响应，最终图可能更平滑，但其表达的是该类别整体证据的时间分布。

对于上采样步骤，我们现已明确说明：由于注意力图对应的是主干网络下采样后的时间长度 `L'`，因此我们通过线性插值将其映射回原始信号长度 `L`，以便与原始 ECG 波形在同一时间轴上对齐展示。以 PTB-XL 主实验为例，该步骤将压缩后的解释图从 `L'=32` 重新对齐到长度为 `L=1000` 的原始时间轴。该步骤仅服务于可视化，不参与额外推理，也不会引入新的诊断信息；最终解释图的真实时间分辨率仍然受主干网络输出长度 `L'` 的限制。相关说明现已补充至主干网络描述、MQAH 模块描述以及解释生成小节，并同步改进了相关图注。
