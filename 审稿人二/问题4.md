# 问题 4：多个查询带来的额外计算开销

## 问题

多个查询很可能会增加计算成本，但论文并未报告训练/推理时间、参数量或内存消耗。

## 回复

我们同意，并已通过新增效率分析来回应这一问题。修订稿现在报告了当前代码库可直接复现的三类效率指标：trainable parameter count、FP32 weight footprint，以及 batch-1 CPU forward latency。与此同时，我们也在正文中明确写出了这种权衡：从单查询对照转向更分解的 MeDeA / MQAH 形式，虽然提升了解释分解能力，并在内部对照中带来有限增益，但同时也会增加参数量和前向计算成本。我们现在将这种权衡表述为一种有代价的设计选择，而不是没有成本的收益；同时也明确说明，这些结果并不是完整的训练时间、FLOPs 或峰值显存 benchmark。
