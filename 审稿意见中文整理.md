## 2. Reviewer 1 中文翻译

审稿人 1 总评翻译：

该稿件提出了用于 ECG 分类的 MeDeA（Multi-disease Decompositional Attention，多疾病可分解注意力）方法，并强调其内在可解释性。论文整体组织较好，研究问题有意义，方法具有一定的新颖性。实验部分也相对全面，同时包含定量和定性分析。总体而言，这是一项有前景的工作；但在论文被考虑接收前，仍有若干问题需要解决。

### 2.1 方法描述仍然不充分

1. 虽然文中说明多标签任务使用 BCE，单标签任务使用 CE，但没有充分解释如何处理类别不平衡问题，尤其是在多标签场景下，不同类别的出现频率差异较大。作者应说明类别权重如何确定，或者是否使用了其他不平衡处理策略。
2. 1D ResNet 主干网络与 MQAH 模块之间的关系还可以描述得更清楚。特别是模块之间特征维度如何匹配，以及对注意力图进行平均和上采样之后会如何影响最终解释，这些都需要更明确说明。

### 2.2 实验公平性和可复现性需要加强

1. Table 3 中对比实验缺少一些重要细节，例如训练设置、模型规模、是否使用数据增强、以及所有方法是否采用相同的数据划分。缺少这些信息后，读者很难判断对比是否完全公平。
2. 对 ECGNet、IMLENet 以及其他基线的比较中，作者应明确是否使用了相同的数据划分，以及是否采用了可比的超参数调优协议。
3. 论文多次强调相较 Transformer 类模型的优势，并提到“可信性和计算可行性”，但文中没有给出参数量、FLOPs、推理时延或显存/内存占用等结果。如果作者希望支撑可部署性的结论，应增加这些分析。

### 2.3 临床适用性和泛化能力讨论需要加强

1. 当前评估主要基于公开的回顾性数据集。论文应更明确地讨论模型在不同医院、不同设备、不同采样率下的鲁棒性，以及模型对高质量标注的依赖程度。

### 2.4 参考文献准确性需要仔细核查

1. 文中将 IMLENet [27] 作为相关对比方法，但参考文献 [27] 似乎对应的是一篇关于船舶滚动轴承故障诊断的论文，而不是 ECG 相关方法。这说明可能存在引用错配或编号错误。作者应认真核查这一问题，并全面检查其他参考文献的准确性。

### 2.5 图表与文献使用还可以进一步改进

1. Figure 1 和 Figure 2 的图注与描述可以更丰富，例如标明特征图维度或注意力输出尺寸，以提升可复现性。
2. Figure 3 当前主要展示的是比例信息。如果能够补充样本数量，或更清楚地解释混淆矩阵是如何构建的，结果会更容易解释，尤其是对少数类而言。
3. 相关工作中引用 arXiv 论文的比例较高。建议尽可能增加更多已正式同行评审的期刊或会议论文，以增强文献综述与实验比较的严谨性。

### 2.6 Reviewer 1 结论

该工作具有一定价值并显示出潜力，但在方法清晰性、实验公平性、计算分析和参考文献准确性方面仍存在若干重要问题。因此建议：**大修**。

## 3. Reviewer 2 中文翻译

审稿人 2 总评翻译：

该稿件提出了一个 Multi-Query Attention Head（MQAH）机制，用于将不同注意力头与不同心脏疾病关联起来。论文中有几个方面仍需要进一步澄清和论证：

1. Figure 1 分辨率较低，MQAH 结构难以辨认。建议替换为高清图或矢量图，以提高清晰度。
2. Multi-Query Attention Head 的技术细节描述不够充分。尤其是多个可学习 query 的生成过程没有解释清楚。虽然文中说 query 是可学习向量，但 Figure 2 看起来又像是由输入特征生成的，这造成了歧义，需要澄清。另外，温度参数 `τ` 的具体取值没有给出。
3. 在 Table 3 中，MSW-TN 在部分指标上优于 MeDeA。作者应分析这一结果，并说明产生性能差距的原因，以及 MeDeA 的优势体现在哪里。
4. 多 query 的引入很可能增加计算开销，但论文没有给出效率分析。建议报告训练时间、推理时间、参数量和内存消耗。
5. 论文假设不同 query 会捕捉不同的形态学模式，但目前缺少足够证据支持这一点。建议提供更全面的可视化结果，例如不同样本上的逐 query 注意力图，以定性展示这种行为。

## 5. 结合代码与现有结果，当前高优先级修改点

### A. 论文实验结论与代码结果要严格对齐
- 仓库现有综合实验报告 `ECG_Classification_Experiment_Report.md` 显示，当前最优模型是 **Inception Baseline**，`Macro F1 = 0.7477`。
- 而现有 MeDeA 相关结果中：
  - `models/saved_models/medea_experiment/cross_validation_summary.json` 的十折测试 F1 大约在 **0.75 左右**
  - `models/saved_models/medea_advanced_se/cross_validation_summary.json` 的平均测试 F1 为 **0.7428**
  - `models/saved_models/medea_single_query_experiment/cross_validation_summary.json` 的测试 F1 也大致在 **0.75 左右**
- 如果论文写的是 “MeDeA 明显优于现有基线” 或 “提出方法达到 SOTA”，这部分必须核对并可能重写。

### B. 模型版本命名与论文主模型要统一
- 代码里同时存在 `MedeA.py`、`MedeA_advanced.py`、`MedeA-singlequery.py`、`MedeA_realmutiquery.py`、多个 ablation 版本。
- 论文必须明确：
  - 主模型到底是哪一个版本
  - 论文主表中的结果来自哪个脚本
  - 消融实验的基线对应哪个 full model
- 否则审稿人很容易质疑复现性和实验严谨性。

### C. 消融实验描述需要补强
- `models/Ablation/run_ablation_experiments.py` 中已经设计了多个消融方向，但报告中含有“参考数据”“如果可用”等占位表述。
- 如果论文里已有消融结论，建议补充：
  - 每个模块移除后的平均 F1/AUC
  - 标准差
  - 每类疾病性能变化
  - 为什么该模块有效的机制解释

### D. 方法创新点需要更清楚地区分“原始版”和“增强版”
- 从代码看，至少存在两条不同技术路线：
  - `models/MedeA.py`：共享 backbone + disease-specific attention heads
  - `models/MedeA_advanced.py` / `models/MedeA_realmutiquery.py`：SE-ResNet backbone + multi-query attention + entropy regularization
- 如果论文把这些创新混在一起写，建议拆成：
  - 核心方法
  - 增强模块
  - 可解释性模块
  - 消融验证

### E. 可解释性部分要避免“只展示图片，缺少定量验证”
- 代码中生成了大量 attention overlay、confusion matrix、class comparison 图。
- 但如果论文只给可视化，没有定量指标，审稿人可能会质疑解释性真实性。
- 建议增加：
  - attention 稀疏性或熵指标
  - 不同类别 attention 差异统计
  - 典型正确/错误案例分析

## 6. 结合当前代码，我对论文和实现的一轮判断

### 6.1 代码里已经有一部分可回应审稿意见的材料

- `models/MedeA.py` 给出了 disease-specific attention head 的基础实现。
- `models/MedeA_advanced.py` 和 `models/MedeA_realmutiquery.py` 已经包含：
  - `SE-ResNet1D` backbone
  - `num_queries` 可学习 query
  - attention entropy regularization
  - 十折交叉验证
  - attention 可视化
- `models/Ablation/` 已经有多个消融版本，这对 rebuttal 很有用。

### 6.2 但代码和论文很可能还存在这些不一致风险

1. 主模型版本不统一。论文如果写的是一个版本，但结果来自另一个脚本，会被质疑可复现性。
2. 现有仓库综合报告显示最优结果来自 `Inception Baseline`，这与“MeDeA 明显领先”的叙事可能冲突。
3. 虽然代码里有 attention entropy 和 multi-query，但论文如果没有把这些公式、维度和训练目标写清楚，就会被认为方法描述不足。
4. 代码里目前没有看到现成的效率统计输出，因此参数量、延迟、显存、FLOPs 需要补跑或补实现。
5. 逐 query 可视化当前还不够系统，需要专门导出而不是只给平均 attention。

## 7. 建议的论文修改方向

### 第一类：方法描述
- 明确输入张量维度、backbone 输出维度、attention 公式、query 数量、每类独立 head 的作用。
- 画一张与最终代码一致的总架构图。
- 统一术语：`single-query`、`multi-query`、`advanced`、`SE-ResNet` 不要混用。

### 第二类：实验设置
- 明确 PTB-XL 预处理、采样率、类别定义、十折划分方式。
- 给出训练轮数、batch size、学习率、patience、随机种子、硬件环境。
- 明确阈值是固定 `0.5` 还是按验证集调优。

### 第三类：结果呈现
- 主结果表中同时列出 MeDeA 与所有 baseline。
- 如果 MeDeA 不是最优，不要硬写成最优；应改成“具有竞争力”并突出可解释性或结构优势。
- 增加均值 ± 标准差，而不是只给单次结果。

### 第四类：消融与讨论
- 用统一数据划分和统一训练策略重跑 full model 与所有 ablations。
- 解释为什么 HYP 类别相对更难。
- 增加失败案例与局限性讨论。
